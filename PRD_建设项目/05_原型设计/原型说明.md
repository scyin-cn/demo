# 原型说明

> 项目：罗布人村寨 AI 接驳车调度系统（管理大屏）
> 版本：五一 MVP 试点版
> 创建时间：2026-04-06

---

## 1. 原型概述

本原型使用 AI Coding 工具生成可交互 HTML，用于：
- 视觉方案确认
- 交互流程验证
- 开发前技术可行性验证

**原型工具**：AI Coding（v0.dev / Claude / 等）
**输出格式**：单个 HTML 文件（含 CSS、JS）
**运行环境**：现代浏览器（Chrome/Firefox/Edge）

---

## 2. 页面清单

| 序号 | 页面名称 | 说明 | 优先级 |
|------|----------|------|--------|
| 1 | 实时监控大屏 | 唯一主页面，包含顶部栏、信息区、地图区、消息区 | P0 |
| 2 | 系统状态抽屉 | 从顶部栏展开，展示各服务健康状态 | P1 |
| 3 | 预测详情弹窗 | 点击预测消息卡片后弹出，展示趋势图/表格 | P1 |
| 4 | 地图浮窗 | 点击车辆/站点标记后弹出，展示详情信息 | P0 |

## 3. 原型设计说明

### 3.1 布局架构

```
┌─────────────────────────────────────────────────────────────┐
│  🔶 顶部栏（64px）                                           │
│  系统标题 | 今日接待 1,247人 | 在线 8/12辆 | 🟢系统正常 10:32:45│
├───────────────┬───────────────────────────────┬─────────────┤
│               │                               │             │
│   信息区       │                               │   消息区     │
│  （左侧25%）   │         地图区                 │  （右侧15%） │
│   320px min   │         （中间60%）            │  200px min  │
│               │                               │             │
│  [车辆][站点]  │                               │  🔔 消息中心 │
│               │     🗺️ 高德地图               │             │
│  ┌─────────┐  │     🚌 车辆图标                │  ┌───────┐  │
│  │🟢 A01   │  │     📍 站点标记               │  │⚠️ 预警 │  │
│  │距东沙山 │  │                               │  │🔥 超限 │  │
│  └─────────┘  │                               │  └───────┘  │
│               │                               │             │
└───────────────┴───────────────────────────────┴─────────────┘
```

### 3.2 颜色方案

| 用途 | 颜色值 | 说明 |
|------|--------|------|
| 主背景 | `#0a0e1a` | 深色 Navy |
| 侧栏背景 | `#111827` | 略浅的 Navy |
| 卡片背景 | `#1f2937` | 卡片/浮窗背景 |
| 主文字 | `#ffffff` | 白色 |
| 次要文字 | `rgba(255,255,255,0.7)` | 70%白 |
| 绿色（正常） | `#10b981` | 正常状态、空载 |
| 蓝色（信息） | `#3b82f6` | 半载、预防预警 |
| 橙色（警告） | `#f97316` | 满载 |
| 红色（危险） | `#ef4444` | 危险状态、补救预警 |

### 3.3 字体方案

- 中文：系统默认（PingFang SC / Microsoft YaHei）
- 数字：DIN Alternate / Roboto（等宽、易读）
- 标题：18px bold
- 正文：14px regular
- 辅助：12px regular

### 3.4 响应式范围

| 断点 | 最小尺寸 | 说明 |
|------|----------|------|
| 大屏 | 1920×1080 | 目标分辨率，最佳显示 |
| 中屏 | 1280×720 | 最小支持分辨率 |
| 小屏 | < 1280×720 | 不保证完全显示 |

## 4. 交互说明

### 4.1 全局交互

| 交互 | 触发 | 效果 |
|------|------|------|
| 时间更新 | 每秒自动 | 顶部栏时间实时更新 |
| 车辆位置刷新 | 每秒自动 | 地图车辆标记平滑移动 |
| 站点人数刷新 | 每秒自动 | 列表数字变化时闪烁 |

### 4.2 顶部栏交互

| 交互 | 触发 | 效果 |
|------|------|------|
| 查看系统状态 | 点击"系统正常" | 右侧滑出抽屉窗，展示各服务状态 |
| 关闭抽屉 | 点击✕或再次点击状态 | 抽屉滑回 |

### 4.3 信息区交互

| 交互 | 触发 | 效果 |
|------|------|------|
| 切换Tab | 点击"车辆"/"站点" | 内容区切换，Tab下划线高亮 |
| 筛选车辆 | 点击筛选按钮 | 列表按状态过滤 |
| 选中车辆 | 点击车辆行 | 行背景高亮，地图车辆放大+居中+浮窗 |
| 选中站点 | 点击站点行 | 行背景高亮，地图站点居中+浮窗 |

### 4.4 地图区交互

| 交互 | 触发 | 效果 |
|------|------|------|
| 点击车辆 | 点击地图车辆图标 | 弹出车辆浮窗，信息区切换车辆Tab并高亮 |
| 点击站点 | 点击地图站点标记 | 弹出站点浮窗，信息区切换站点Tab并高亮 |
| 点击空白 | 点击地图空白处 | 关闭所有浮窗，取消列表高亮 |

### 4.5 消息区交互

| 交互 | 触发 | 效果 |
|------|------|------|
| 查看预警 | 点击预警卡片"查看" | 地图定位到对应站点，站点标记闪烁 |
| 查看预测 | 点击预测卡片 | 弹出蒙版+详情弹窗（趋势图/表格） |
| 关闭弹窗 | 点击✕或蒙版 | 弹窗关闭 |

## 5. 动画说明

| 动画 | 场景 | 参数 |
|------|------|------|
| 呼吸动画 | 行驶中车辆图标 | 1.5s ease-in-out infinite |
| 慢闪 | 拥挤站点 | 2s steps(1) infinite |
| 快闪+波纹 | 危险站点 | 1s steps(1) infinite |
| 脉冲 | 选中车辆 | 1.5s ease-out infinite |
| 抽屉滑动 | 系统状态抽屉 | 300ms ease |
| 弹窗淡入 | 详情弹窗 | 200ms ease |
| 列表高亮 | 选中行 | 150ms ease |

## 7. AI Coding 实现

详细实现指南见同目录 `AI Coding 实现指南.md`

**关键要点**：
- 使用高德地图 JS API（申请临时 key）
- 使用 CSS 变量统一管理颜色和尺寸
- 使用 Flexbox 布局确保响应式
- 模拟数据用 JavaScript 对象硬编码
- 动画使用 CSS @keyframes
