# 产品需求文档（PRD）

> 项目：罗布人村寨 AI 接驳车智能调度系统（管理大屏）
> 版本：五一 MVP 试点版
> 创建时间：2026-04-06
> 文档状态：待评审

---

## 1. 文档概述

### 1.1 项目背景
新疆罗布人村寨景区五一期间客流高峰，需从人工经验调度升级为数据驱动调度，提升游客体验。

### 1.2 产品定位
- **用户**：调度员老王（日常值守）、运营负责人李经理（盯盘决策）
- **场景**：调度室大屏实时监控
- **核心功能**：车辆实时位置、站点排队监控、智能预警、客流预测
- **边界**：本版本只做展示+预警，不做自动调度（用对讲机沟通）

### 1.3 术语表

| 术语 | 说明 |
|------|------|
| 主摄像头 | 站点候车区摄像头，统计当前排队人数 |
| 前置摄像头 | 站点上游摄像头，统计即将到达人数 |
| 预防性预警 | 涌入+车少时触发，提前派车 |
| 补救性预警 | 排队超限时触发，紧急派车 |
| 满载率 | 当前人数/额定人数×100% |
| 在线车辆 | GPS 10分钟内更新算在线 |

## 2. 功能详述

### 2.1 顶部栏模块

#### 功能1：今日接待人数展示
- **输入**：绿云系统API（今日人数、昨日人数）
- **处理**：每30秒刷新，计算增减百分比
- **输出**：大字号显示"1,247"，小字"+12%"（绿色正/红色负）
- **异常**：API超时显示"—"

#### 功能2：在线车辆数展示
- **输入**：车辆GPS状态（最后更新时间）
- **处理**：每秒轮询，10分钟内更新算在线
- **输出**："8/12辆"（在线/总数）
- **异常**：GPS断开显示"—/12辆"

#### 功能3：系统健康状态展示
- **输入**：7项核心服务心跳（GPS/高德/绿云/海康/车载/后端/YOLO）
- **处理**：每30秒健康检查
- **输出**：🟢正常/🟡预警/🔴异常，点击展开抽屉窗
- **抽屉窗**：服务列表+状态灯+断开时长+最后响应时间+错误信息+【重试连接】

### 2.2 信息区模块

#### 功能4：车辆列表展示
- **输入**：车辆GPS、YOLO计数
- **处理**：每秒刷新，计算满载率和最近站点
- **排序**：在线优先→满载率高的优先
- **输出**：状态灯+车牌+满载率+位置（距XX站点X.Xkm）

#### 功能5：车辆列表筛选
- **筛选项**：全部/行驶中/静止/离线
- **交互**：点击后列表实时过滤

#### 功能6：车辆行点击交互
- **联动**：列表行高亮→地图车辆居中+放大+脉冲动画→弹出浮窗

#### 功能7：站点列表展示
- **输入**：主/前置摄像头YOLO计数
- **处理**：每秒刷新，判定状态（<20正常/20-40拥挤/>40危险），计算预计等待
- **输出**：状态灯+名称+排队人数+预计等待
- **淡旺季**：神女湖淡季显示"未开放"

#### 功能8：站点行点击交互
- **联动**：列表行高亮→地图站点居中+闪烁动画→弹出浮窗

### 2.3 地图区模块

#### 功能9：高德地图接入
- **配置**：中心坐标、缩放14、深色主题、隐藏缩放按钮
- **显示**：景区边界+4条接驳路线（虚线）

#### 功能10：车辆标记展示
- **图标**：32×32px巴士俯视图，车牌标签
- **颜色**：空载绿/半载蓝/满载橙（发光）/离线灰（虚线）
- **动画**：行驶中呼吸动画（1.5s）
- **刷新**：每秒平滑移动

#### 功能11：车辆标记点击交互
- **高亮**：放大至48px+金色发光+脉冲动画
- **联动**：信息区切到车辆Tab+高亮对应行

#### 功能12：站点标记展示
- **标记**：人数标签（黑底白字）+状态圆点+名称
- **动画**：拥挤慢闪（2s）/危险快闪+波纹（1s）
- **淡旺季**：未开放透明度50%

#### 功能13：地图浮窗展示
- **车辆浮窗**：车牌/状态/位置/载客（4/20人，20%）
- **站点浮窗**：名称/状态/排队/即将到达/预计等待/最近车辆（2辆）
- **位置**：标记右侧20px，边界检测（超界则显示在左侧）

### 2.4 消息推送区模块

#### 功能14：日预测消息生成
- **触发**：每日18:00
- **算法**：历史7天同日平均×天气系数×节假日系数
- **输出**：预计接待+推荐车辆+【查看详情】（24小时趋势图）
- **生命周期**：24小时后归档

#### 功能15：周预测消息生成
- **触发**：每周一18:00
- **输出**：7天每日预测+推荐车辆+【查看详情】（7天表格）
- **生命周期**：7天后替换

#### 功能16：预防性预警生成
- **触发**：涌入≥50人（可配置）+可用车辆<2辆
- **输出**：蓝色卡片，【查看】按钮
- **声音**："叮"一声
- **生命周期**：1小时后消失

#### 功能17：补救性预警生成
- **触发**：排队>40人（可配置）
- **输出**：红色卡片，【查看】按钮
- **声音**："滴滴滴"三声（更急促）
- **生命周期**：1小时后消失

#### 功能18：消息静音开关
- **位置**：消息区标题栏右侧（🔔/🔕）
- **功能**：点击后所有预警声音关闭

#### 功能19：预测详情弹窗
- **触发**：点击预测卡片
- **效果**：半透明蒙版+居中弹窗（600×400px）
- **关闭**：点击✕或蒙版（200ms淡出）

## 3. 非功能性需求

### 3.1 性能
- 页面加载：<2秒（1920×1080，Chrome 90+）
- 车辆位置延迟：<1.5秒
- 并发用户：≥10人

### 3.2 兼容性
- 浏览器：Chrome 90+, Firefox 88+, Edge 90+
- 分辨率：1920×1080（最佳）/ 1280×720（最小）

### 3.3 安全性
- 传输：HTTPS强制
- 访问：IP白名单
- 权限：只读（大屏不做修改）

### 3.4 可靠性
- 可用性：≥99.5%
- 故障恢复：<5分钟
- 降级：GPS/摄像头离线仍可运行

### 3.5 可维护性
- 日志：操作日志+错误日志，保留30天
- 告警：核心服务异常短信/邮件通知
- 配置：阈值可配置（预警阈值、刷新频率）

## 4. 验收标准与测试要点

### 4.1 功能验收（18项）

| 模块 | 验收项 | 标准 |
|------|--------|------|
| 顶部栏 | 接待人数/在线车辆/健康状态 | 数据一致，状态正确 |
| 信息区 | Tab切换/筛选/满载率/站点状态 | 响应<200ms，计算准确 |
| 地图区 | 加载<2s/标记颜色/浮窗/联动 | 动画流畅，位置准确 |
| 消息区 | 预测生成/预警触发/声音/生命周期 | 定时准确，声音区分 |

### 4.2 非功能验收

| 项 | 标准 | 方法 |
|----|------|------|
| 加载时间 | <2秒 | Lighthouse |
| 并发10用户 | 正常刷新 | JMeter |
| 7×24小时 | 无内存泄漏 | 持续运行 |
| 数据准确性 | 位置<50m，人数误差<10% | 实测对比 |

### 4.3 UAT
- **参与人**：老王（调度员）、李经理（负责人）、技术1人
- **场景**：日常监控/高峰预警/夜间值守/交接班
- **通过标准**：功能100%+非功能90%+UAT无阻塞
