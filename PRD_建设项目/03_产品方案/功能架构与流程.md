# 功能架构与流程

> 项目：罗布人村寨 AI 接驳车智能调度系统（管理大屏）
> 版本：五一 MVP 试点版
> 更新时间：2026-04-06

---

## 1. 信息架构总览

### 1.1 页面区域划分（已确认）

| 区域 | 名称 | 核心功能 | 占比 | 位置 |
|------|------|----------|------|------|
| ① | 信息区 | Tab 切换：车辆列表 / 站点列表 | 约 25% | **左侧** |
| ② | 地图区域 | 高德地图 + 车辆实时位置 + 站点标记（默认全景，无缩放功能） | 约 60% | **中间** |
| ③ | 消息推送区 | 预测信息 + 实时预警（无派车按钮） | 约 15% | **右侧** |
| ④ | 顶部栏 | 全局状态 + 今日运营概览 | 约 60px 高 | 顶部 |

## 2. 各区域功能概要

### 2.1 区域①：地图区域（方案B：站点邻近）

**核心定位**：态势感知主视觉区

**方案说明**：
- 不判断车辆具体在哪条路线上、往哪个方向行驶（避免路线重叠判断错误）
- 只计算车辆GPS到各站点的直线距离，显示"距离XX站点XX米"
- 调度员结合地图视觉位置，自行判断车辆状态

**地图底图**：
- 高德地图默认样式（不做自定义主题）
- 显示范围：景区全貌（包含4个站点和接驳路线）
- 控件：隐藏缩放按钮（不做缩放功能）、隐藏鹰眼、仅保留比例尺
- 路线显示：4条接驳路线用浅色虚线标注

**车辆图标设计**：

| 属性 | 设计 |
|------|------|
| **图标** | 简化巴士俯视图（🚌 风格化） |
| **大小** | 正常状态 32×32px，高亮状态 48×48px |
| **车牌显示** | 图标下方，白色背景+深色字，字号12px |
| **行驶状态** | 静止=静态图标；行驶中=轻微"呼吸"动画 |
| **高亮状态** | 放大+金色发光边框+脉冲动画 |

**车辆满载率颜色**：

| 状态 | 图标颜色 | 边框 | 说明 |
|------|----------|------|------|
| **空载** | 🟢 绿色 | 细边 | 载客 < 30% |
| **半载** | 🔵 蓝色 | 细边 | 载客 30%-70% |
| **满载** | 🟠 橙色 | 粗边 | 载客 > 70% |
| **离线** | ⚪ 灰色 | 虚线边 | GPS 10分钟无更新 |

**站点标记设计**：

| 状态 | 圆点颜色 | 排队人数标签 | 动画 | 触发条件 |
|------|----------|--------------|------|----------|
| **正常** | 🟢 绿色 | 白色背景 | 无 | 排队 < 20人 |
| **拥挤** | 🟡 黄色 | 黄色背景 | 慢闪（2秒/次） | 排队 20-40人 |
| **危险** | 🔴 红色 | 红色背景 | 快闪+扩散波纹 | 排队 > 40人 |

**站点显示**：
- 站点名称（游客中心、阿不旦广场、东沙山、神女湖）
- 当前排队人数（主摄像头统计，白色数字深色底）
- 淡旺季切换：神女湖站点淡季自动隐藏

**车辆详情浮窗**：

```
┌─────────────────────┐
│ 🚌 车辆 A01      [×] │
├─────────────────────┤
│ 状态: 🟢 空载 行驶中 │  ← 满载率+行驶状态
│                     │
│ 当前位置:           │
│ 距离东沙山 1.2km    │  ← 距最近站点距离
│                     │
│ 载客: 4/20人 (20%)  │
└─────────────────────┘
```

**浮窗字段说明**：
- ✅ 车牌号、状态（满载率+行驶/静止）
- ✅ 位置（距离最近站点XX米）
- ✅ 载客情况（当前/额定+百分比）
- ❌ 行驶方向、目的地、预计到达时间、历史轨迹、联系司机

**站点详情浮窗**：

```
┌─────────────────────┐
│ 📍 东沙山        [×] │
├─────────────────────┤
│ 状态: 🔴 危险       │
│ 当前排队: 48人      │  ← 主摄像头
│ 即将到达: +12人     │  ← 前置摄像头（5分钟内）
│ 预计等待: 15分钟    │  ← 基于排队和车辆计算
│                     │
│ 最近车辆:           │
│ 🚌 A03 (1.2km)      │
│ 🚌 A05 (3.5km)      │
└─────────────────────┘
```

**关键交互**：
- 点击信息区车辆行 → 地图车辆图标高亮（放大+金色边框）+ 自动居中 + 弹出浮窗
- 点击信息区站点行 → 地图站点标记高亮（红色波纹）+ 自动居中 + 弹出浮窗
- 点击地图车辆/站点 → 信息区自动切换对应Tab + 高亮对应行
- 点击消息区预警 → 地图自动定位到相关站点/车辆
- 点击地图空白处 → 关闭所有浮窗

**数据刷新频率**：
- **车辆位置：每秒刷新**（平滑移动过渡动画）
- **站点人数：每秒刷新**（数字变化时闪烁一次）
- 车辆状态：每30秒刷新（颜色变化时过渡动画）

### 2.2 区域②：信息区（修订版）

**核心定位**：数据导航与快速定位

**布局**：侧边栏，Tab 切换，**无视频面板**

#### Tab 1：车辆列表

**刷新频率**：位置信息 **每秒刷新**（保证实时性）

| 字段 | 说明 | 视觉 |
|------|------|------|
| **车牌号** | 如 A01、A02 | 左对齐，加粗 |
| **状态** | 🟢行驶中 / ⏸️静止 / ⚪离线 | 图标+文字 |
| **满载率** | 20%、65%等 | 右侧，颜色区分（绿/蓝/橙） |
| **位置** | "距东沙山1.2km" | 灰色小字，第二行 |

**视觉样式**：
```
┌─────────────────────┐
│ 🟢 A01        20%   │  ← 绿色=空载行驶中
│ 距东沙山1.2km       │  ← 每秒更新
├─────────────────────┤
│ ⏸️ A02        45%   │  ← 蓝色=半载静止
│ 距阿不旦0.8km       │
├─────────────────────┤
│ ⚪ A04       离线   │  ← 灰色=离线
│ GPS信号丢失         │
└─────────────────────┘
```

**交互**：
- 点击车辆行 → 地图对应车辆高亮 + 自动居中 + 浮窗
- 顶部筛选栏：全部/行驶中/静止/离线
- 排序：在线车辆优先，其次按满载率（高的在前）

**特殊状态**：
- 高亮选中：背景浅蓝色
- 满载预警：满载率>70%，边框变色
- 离线过久：离线>30分钟，降低不透明度

#### Tab 2：站点列表

**字段**：

| 字段 | 说明 |
|------|------|
| **站点名** | 游客中心、阿不旦广场等 |
| **状态灯** | 🟢🟡🔴 圆点 |
| **当前排队** | 48人（主摄像头） |
| **即将到达** | +12人（前置摄像头） |
| **预计等待** | 15分钟 |

**注意**：
- 站点列表无调度按钮（调度员用对讲机沟通）
- 站点列表无视频功能（另一平台已有）
- 无视频面板

**淡旺季**：神女湖淡季显示"未开放"或隐藏

**视觉样式**：
```
┌─────────────────────┐
│ 🟢 游客中心          │
│ 排队12人  预计3分钟  │
├─────────────────────┤
│ 🟡 阿不旦广场        │
│ 排队35人  +8人即将   │
│ 预计8分钟            │
├─────────────────────┤
│ 🔴 东沙山            │
│ 排队48人  +12人即将  │  ← 危险状态
│ 预计15分钟           │     调度员用对讲机沟通
└─────────────────────┘
```

### 2.3 区域③：消息推送区（修订版）

**核心定位**：智能预警信息入口

**布局**：右侧栏，卡片式列表

**本版本范围**：
- 只做预测消息和预警消息
- 不做调度工单
- 不做调度建议功能
- 预警卡片无【派车】按钮（调度员用对讲机沟通）

#### 消息类型

| 类型 | 触发条件 | 内容 | 频率 | 操作 |
|------|----------|------|------|------|
| **日预测** | 每日 18:00 自动生成 | 明日预测客流 + 推荐车辆数 | 每天 1 条 | [查看详情] |
| **周预测** | 每周一 18:00 自动生成 | 下周每日预测 + 每日推荐车辆 | 每周 1 条 | [查看详情] |
| **预防性预警** | 前置检测50人涌入 + 可用车辆<2辆 | 东沙山即将涌入50人，当前可用车辆1辆 | 实时 | [查看] |
| **补救性预警** | 主摄像头排队 > 40人 | 阿不旦排队48人，超过阈值 | 实时 | [查看] |

#### 消息卡片样式

**日预测**：
```
┌─────────────────────┐
│ 📊 明日客流预测      │
│ 预计接待: 1,500人   │
│ 推荐车辆: 10辆      │
│          [查看详情] │
└─────────────────────┘
```

**预防性预警（蓝色）**：
```
┌─────────────────────┐
│ ⚠️ 预防性预警        │
│ 东沙山即将涌入50人  │
│ 当前可用车辆: 1辆   │
│ 建议增派车辆        │
│          [查看]     │  ← 仅查看，无派车按钮
└─────────────────────┘
```

**补救性预警（红色）**：
```
┌─────────────────────┐
│ 🔥 排队超限预警      │
│ 阿不旦排队48人      │
│ 已超过阈值(40人)    │
│ 建议立即派车        │
│          [查看]     │  ← 仅查看，调度员用对讲机沟通
└─────────────────────┘
```

#### 交互流程

```
预警消息出现
    ↓
调度员看到预警（声音+视觉提示）
    ↓
点击【查看】→ 地图自动定位到站点
    ↓
调度员观察地图车辆位置
    ↓
调度员用对讲机呼叫司机派车
    ↓
（系统不记录派车操作，本版本不做）
```

#### 详情弹窗（点击预测卡片）

- 背景半透明蒙版
- 弹窗居中
- 日预测：显示明日各时段预测人数趋势
- 周预测：显示7天预测表格

#### 消息生命周期

| 类型 | 保留时长 | 说明 |
|------|----------|------|
| 日预测 | 24小时 | 次日自动归档 |
| 周预测 | 7天 | 下周自动生成时替换 |
| 预警 | 1小时 | 自动消失（或手动关闭）|

### 2.4 区域④：顶部栏

**核心定位**：全局运营状态一眼掌握

**展示内容：**

| 指标 | 数据来源 | 计算逻辑 | 展示方式 |
|------|----------|----------|----------|
| **今日接待人数** | 绿云系统 | 闸机入园人数累计 | 大数字 + 较昨日增减百分比 |
| **在线车辆数** | 车辆 GPS | GPS 信号正常 + 10 分钟内有位置更新的车辆 | "8/12辆"（在线/总数） |
| **系统健康状态** | 各子系统心跳检测 | 所有接口正常=🟢健康，任一故障=🔴异常 | 状态灯 + 点击看详情 |

**系统健康状态判断逻辑：**

| 检测项 | 检测方式 | 判定标准 | 优先级 |
|--------|----------|----------|--------|
| GPS 服务 | 定时发送测试坐标 | 5 秒内返回正常 | 核心 |
| 高德地图 API | 调用地图加载接口 | HTTP 200 且返回正常 | 核心 |
| 绿云数据接口 | 调用闸机人数查询 | 返回数据且时间戳正常 | 核心 |
| 海康摄像头接口 | 调用视频流检测 | 能获取视频帧 | 核心 |
| 车辆车载系统 | 轮询车辆心跳 | 最近 1 分钟内有心跳 | 核心 |
| 自身后端服务 | 健康检查接口 | 数据库连接正常 | 核心 |
| YOLO 算法服务 | 调用人数统计接口 | 返回正常结果 | 非核心 |

**状态显示：**
- 🟢 **健康**：所有核心服务正常
- 🟡 **预警**：非核心服务异常（如 YOLO 服务短暂中断）
- 🔴 **异常**：任一核心服务故障

**交互：**
- 点击"系统状态" → 右侧抽屉窗展示各服务状态（绿灯正常/红灯故障）
- 异常时有声音提示（可关闭）
- 故障服务显示"已断开 2 分钟"等时间提示

**设计要点：**
- 背景与主界面区分（深色或品牌色）
- 数字用大号字体，单位小字
- 刷新频率：30 秒

## 3. 跨区域交互逻辑

### 3.1 信息区 → 地图区

```
信息区点击车辆/站点
        ↓
地图区对应元素高亮（发光边框）
        ↓
地图自动平移，使该元素居中显示
        ↓
弹出详情浮窗（车辆详情/站点详情）
```

### 3.2 消息区 → 地图区

```
消息区点击预警消息【查看】
        ↓
地图自动定位到相关站点
        ↓
站点标记闪烁（红色快闪3次）
```

### 3.3 地图区 → 信息区

```
地图区点击车辆/站点
        ↓
信息区自动切换到对应 Tab（车辆/站点）
        ↓
对应行高亮显示（背景色变化）
```
